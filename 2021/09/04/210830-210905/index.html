학습내용

SQL

실습 환경

  • Aws ec2 인스턴스 사용
  • 인스턴스내에 Mysql설치(내부패스워드, 외부접근권한, 외부패스워스 설정)
  • Local PC에 Workbench를 설치하여 인스턴스의 mysql과 연동

실습 내용

  • 기초 데이터인 world, sakila 데이터베이스를 기본으로 학습
  • CRUD를 활용하여 sql학습
  • 여러 데이터베이스/테이블을 사용 및 관리를 위한 설정 학습(Foreign Key 등 제약조건)
  • 가상환경을 활용한 백업 실습(Hot/logical backup, Cold/physical backup)
  • Replication을 통해 서버간 데이터 연동 실습

mongodb

실습 환경

  • Aws ec2 인스턴스 사용
  • 인스턴스내에 mongodb설치(mysql설치때와 유사하게 세팅)
  • Local PC에 Robo 3T를 설치하여 인스턴스의 mongodb와 연동

실습 내용

  • mongodb의 특성을 위주로 기초적인 문법 학습
  • python과 연동해서 사용하는 이론학습

정리

SQL

  • MySql은 데이터베이스, 테이블 관리가 섬세해서 원하는 작업을 수행하기까지가 어렵다
  • 다만, 출력에 특화 되어있는 프로그램인 만큼, 원하는만큼 자유롭게 조작하여 데이터 추출에 용이히다

Mongodb

  • SQL과 반대로 입력에 특화된 시스템이다.
  • sql 테이블에 해당하는 컬렉션끼리 조작이 되지 않지만, 데이터를 입력 받을때 별다른 제약이 없다는 점에서 짧은 시간에 많은 데이터가 발생하는 상황에 사용하기 좋은 프로그램이다.

(- tmux를 활용해 협업을 하는 방법을 배웠다.tmux내 윈도우 분할 기능이 잘 안된다)

이번주 학습 총평

처음 다뤄보는 프로그램이지만, 문법이 직관적이라 그런지 학습하는데 크게 어렵지 않았다.
심화 강의를 재수강하며 데이터를 가져오고 테이블을 세팅하는 방법의 연습이 필요해보인다
컴퓨터로 작업을 할 때, 작업을 수행하는게 전부가 아니라는 것을 점점 알아가고 있다.
생각보다 내가 목표하는 작업에 대한 준비는 시간 소요가 적어보인다. 다만, 환경설정, 오류에 대한 대처가 중요해보인다.
강사님께서 말씀하신 것처럼, 체감삼 업무의 80%는 오류를 해결하는데 있다고 한다.
진도를 따라가야하기 때문에, 강사님께 물어보는 경우가 생기지만 최대한 혼자서 해결해보는걸 연습하는 중이다.
오류메세지를 읽어보고, 구글링을 한 것만으로 해결되는 경우가 굉장히 많았다.
러버덕 디버깅에 대한 중요함을 강의 당일 깨달았다. 그 수많은 시도를 하면서 commit을 해볼 생각을 왜 못했을까
하지만, 조금이라도 고민하고 고생한 끝에 많은 것을 배울 수 있었다.
검색을 하는 방법, 공유할만한 사이트 등등 어떤 일을 할지, 어떻게 할지 감이 아직 없는 상태지만 그래도 실무에 한 발 더 다가간 기분이 든다
난생 처음까진 아니지만, 내가 스스로 복습을 하고 있는 요즘이 참 새롭다. 없던 욕심도 생기고 20대를 시작하면서 내가 정말 느끼고 싶어했던, 무언가에 집중하고 열심히 했다! 하는 느낌을 받기 시작했다.
아무래도 그냥그냥 살아온 시간이 벌써 약 10년이 되어가는 시점이라 다시 그 느낌을 꺼내오긴 힘들어보이지만, 시간과 노력이 해결주리라 본다.
신기하게도 30이 다 되어가는 지금 시점에 무언가를 다시 시작하고 있는데, 후회나 걱정이 전혀 생기지 않고 있다.
그렇게 밖을 싸돌아 다니고 혼자 생각하는 시간을 가질때는 무언가 답답했는데, 매일 10시간씩 같은 자리에 있는 지금이 더 개운한 느낌이 든다.

열심히 하자. 내가 시작한거니까 이러나 저러나 어떻게든 살겠지만, 그래도 조금은 더 욕심을 내보자!
이번주도 수고했다.
